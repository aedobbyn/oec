UN COMTRADE

Here we use the Tidy Data principles exposed by Hadley Wickham in the Journal of Statistical Software.

The difference between this and the raw data is that here we reorganized the information providing an exports_fob_usd column (in FOB dollars) and an imports_cif_usd column (in CIF dollars) instead of a Trade Flow column with labels “Imports” and “Exports” and a Trade Value (US$) column that contains both FOB and CIF dollars. Also we provide a column marker with labels as follow:

  1. The reported values are ok (\(CIF/FOB > 1\))
  2. The reported values had to be replaced with counter reported values (\(CIF/FOB = 1\))
  3. The reported values are incomplete (\(CIF/FOB < 1\))

In the case of missing data we checked counter reported numbers, so that in the cases that country \(i\) reported 0 or null exports of product \(p\) to country \(j\) we replaced those values with what country \(j\) reported as imports of product \(p\) from country \(i\) when it was possible.

    Scripts

    If you want to run this scripts in your server create an RStudio project put these scripts in your project folder:

    • Set up everything
    • Create folders
    • Download scripts

    Then run 0_set_up_everything.R, the folders structure will be replicated and the scripts will be downloaded to their respective folders. After that you’ll be ready to run the different scripts of this projects to convert and clean the data yourself.
